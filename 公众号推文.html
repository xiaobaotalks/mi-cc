终端里的 AI 编程助手，不只是聊天那么简单

mi-cc 发布 v2.0：四层记忆 + 多模型故障转移 + MCP Server

写在前面

用过 Cursor、Claude Desktop 的朋友应该都有这个体验：AI 助手确实能写代码，但聊久了上下文就丢了，API 一报错整个对话就断了，而且每次重启都得从头开始。

如果有一个工具，能把对话记忆像游戏存档一样保存，API 出问题时自动切换到备用模型，还能接入你常用的 AI 客户端作为专业编程助手——这就是 mi-cc 正在做的事。

mi-cc 是什么

mi-cc 是一个运行在终端里的 AI 编程助手。它默认对接国产大模型（小米 MiMo），同时也兼容 DeepSeek、月之暗面、智谱 GLM 等任意 OpenAI 协议通道。

和常见的 AI 聊天工具不同，mi-cc 从设计之初就考虑了两个核心场景：

场景一：本地终端编程助手

直接在命令行里和 AI 对话，让它帮你读代码、改文件、跑命令、做重构。对话过程中自动保存进度，即使电脑重启也能接着上次的位置继续。

场景二：MCP Server 接入主流 AI 客户端

通过 --mcp 参数启动，mi-cc 可以作为 MCP Server 被 OpenClaw、Cursor、Claude Desktop 等宿主调用。宿主获得的不只是"读文件""写文件"这样的原子操作，而是完整的编程任务执行能力——自主规划、多轮工具调用、自动压缩上下文、任务断点续跑。

四层记忆：对话不会白聊

mi-cc 的记忆系统分四层，每一层解决不同的问题：

层级 作用 解决什么问题
checkpoint 当前会话检查点 重启后自动恢复对话位置
MEMORY 跨会话项目记忆 长期记住项目架构、编码规范
notes 临时笔记 记录待办、调试思路
history 完整对话历史 随时回溯之前的讨论

实际使用中的感受是：早上和 mi-cc 讨论了一个架构方案，下午打开终端，它还记得上午聊到哪了，甚至记得你说过"这里先用占位符，后面再优化"。

多模型故障转移：一个 API 挂了，自动换下一个

配置多组 API Key 后，mi-cc 会在后台维护一个 Provider 列表。当主模型返回 401（Key 失效）、429（速率限制）或 5xx（服务异常）时，自动切换到下一个可用模型，整个过程对用户透明。

对于需要长时间运行的复杂任务（比如批量重构、代码审查），这个功能意味着你不需要守在电脑前盯着，即使某个模型临时不可用，任务也能继续执行。

分层摘要压缩：聊再久也不丢上下文

和 AI 聊久了，上下文窗口总会满。mi-cc 的处理方式是分三档自动压缩：

  • 60% 时告警，让你知道快满了
  • 80% 时把早期对话压缩成摘要，保留最近 5 轮原文
  • 95% 时紧急合并，保留最近 3 轮

压缩后的摘要会持久化到本地，重启后仍然有效。这意味着你可以和 mi-cc 聊上几十轮，它依然记得项目背景和你之前的决策。

MCP Server：让 Cursor/Claude 获得"完整编程能力"

这是 mi-cc 区别于普通 MCP 工具的核心差异。

普通 MCP 工具暴露给宿主的能力通常是原子操作：读文件、写文件、执行命令。宿主 AI 需要自己去规划"先读哪个文件、再改哪行代码"。

mi-cc 暴露的 agent_execute 工具则不同。它内部包含完整的 Agent 循环：接收任务、规划步骤、调用工具、处理结果、压缩上下文。宿主只需要说"帮我修复这个类型错误"，mi-cc 会自动完成读取、分析、修改、验证的全流程。

接入方式很简单，在 OpenClaw 的配置里加几行就行：

{
  "mcpServers": {
    "mi-cc": {
      "command": "mi-cc",
      "args": ["--mcp"],
      "env": { "API_KEY": "your_key" }
    }
  }
}

技能库：越用越懂你

mi-cc 支持从对话历史中自动蒸馏技能。比如你和它讨论过几次"如何优雅地处理 TypeScript 错误"，执行 /distill 后，这段经验就会被提取成技能，写入 skill-lib.md。

以后当你提到"重构""类型安全"这类关键词时，mi-cc 会自动把相关技能注入到 System Prompt 中，回答质量明显提升。

项目索引：代码库的智能搜索引擎

/index 命令会扫描项目中的 ts/js/json/md 文件，提取函数定义、导入导出关系，建立索引。之后你可以用 /ask 提问：

"这个项目中的 handleToolCalls 函数是做什么的？"

mi-cc 会基于索引找到相关文件，调用 LLM 给出准确的回答。对于接手新项目或回顾自己几个月前写的代码，这个功能非常实用。

三步上手

# 1. 克隆安装
git clone https://github.com/xiaobaotalks/mi-cc.git
cd mi-cc && npm install

# 2. 配置 API Key
cp .env.example .env
# 编辑 .env，填入 API_KEY

# 3. 启动
npm run dev

配置支持小米 MiMo、DeepSeek、月之暗面、智谱 GLM、腾讯混元、百度千帆、阿里通义千问等国产模型，也支持 OpenAI 兼容通道。

写在最后

mi-cc 的定位不是一个"更好的聊天工具"，而是一个"能长期陪伴的编程助手"。它解决的是实际开发中的痛点：对话丢失、API 不稳定、上下文超限、重复解释项目背景。

如果你经常需要在终端里和 AI 协作写代码，或者想让 Cursor/Claude 获得更专业的编程能力，可以试试看。
